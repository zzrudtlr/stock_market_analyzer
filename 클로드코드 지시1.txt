1. 관심 종목 선정 사유 표시

“오늘의 관심 종목” 목록에서 각 종목별 선정 사유를 함께 표시해줘.

표시 방식:
- signal_reason 컬럼을 기본 표시
- 단기 관심 종목은 아래 기준을 요약해서 사유 표시:
  - 최근 5일/20일 수익률
  - 거래량 증가율
  - momentum_score
  - volume_score
  - risk_score
  - final_signal

- 장기 관심 종목은 아래 기준을 요약해서 사유 표시:
  - 20일/60일 수익률

  - relative_strength
  - trend_score
  - risk_score
  - final_signal

예시 문구:

추가 단계: 관심종목 자동 선정 기능

기존 프로젝트 구조를 유지하면서 관심종목 자동 선정 기능을 추가해줘.

기능:
- stock_analysis_results 분석 결과를 기준으로 오늘의 관심종목을 자동 선정
- 단기 관심종목 그룹과 장기 관심종목 그룹을 watchlist_groups에 upsert
- 선정된 종목은 watchlist_items에 group_id + stock_code 기준 upsert
- memo 컬럼에는 선정 사유를 저장
- 선정 사유는 stock_analysis_results.signal_reason을 우선 사용
- signal_reason이 없으면 지표 기준으로 참고용 사유 문구 생성

선정 기준 예시:
- 단기 관심종목:
  - final_signal = '강세 관심'
  - momentum_score 양호
  - volume_score 양호
  - risk_score 과도하지 않음

- 장기 관심종목:
  - final_signal IN ('강세 관심', '추세 유지')
  - ma20 >= ma60 또는 ma60 >= ma120
  - relative_strength 양호
  - trend_score 양호
  - risk_score 과도하지 않음

주의:
- 매수 추천 표현 금지
- 수익 보장 표현 금지
- 모든 저장은 upsert
- 기존 DB 구조 변경 금지
- DROP TABLE 금지
- ALTER TABLE 금지
- 컬럼명 변경 금지
